App Machine — 5 Parallel Regions

The app.machine runs as a single XState v5 actor with five orthogonal parallel regions. Each region tracks an independent concern.

overlay
closed → person | company | vs | gold
Active entity overlay panel
search
idle ↔ active
Workstation search palette state
auth
checking → authenticated | unauthenticated
Currently stub. Real OAuth in Sprint 2.
focus
idle ↔ focused
Cinematic focus mode for entity zoom
atlasView
globe | graph (network)
Active visualization layer

AppContext shape

interface AppContext { selectedEntityRef: EntityRef | null; searchQuery: string; user: AuthUser | null; comparisonRefs: { a?: EntityRef; b?: EntityRef }; pendingURLState: AppURLState | null; // ... region-specific fields }

Key guards

  • urlActuallyChanged — uses fast-deep-equal to prevent URL-sync loops between TanStack Router and machine state
  • isGoldCompanyClick — discriminates Gold overlay click from regular company entity click on globe
  • hasPendingURLState — gates URL-driven transitions during boot

Named actions

  • clearOverlay — closes any overlay, clears selectedEntityRef, dispatches CLEAR_ARCS
  • navigateHome — resets to / index route, clears all parallel region state
  • dispatchClearArcs — fire-and-forget event to deck.gl globe to remove ArcLayer instances
  • syncURLToMachine — receives URL_CHANGED events from router, hydrates context

Sprint 2 review: Currently 5 parallel regions. ADR-0001 sets soft cap at 4. If a 6th region is proposed in Sprint 2, refactor focus into overlay substate first.

03

Routing

Code-based TanStack Router (no file-based plugin). Two routes total. Search params Zod-validated. URL is derived state — components never call router.navigate() directly.

PathComponentSearch params
/IndexRoutenone
/workstationAppShelloverlay, id, a, b, q
const workstationSearchSchema = z.object({ overlay: z.enum(['person', 'company', 'vs', 'gold']).optional(), id: z.string().optional(), a: z.string().optional(), // comparison left b: z.string().optional(), // comparison right q: z.string().optional(), // search query });

The router is the source of truth for URL state. The machine receives URL_CHANGED events from a router subscription and reconciles its parallel region state via syncURLToMachine. The urlActuallyChanged guard breaks the cycle: URL_CHANGED → syncURLToMachine → action calls navigate → router emits → guard rejects identical state.

Architectural Decisions — ADR-0001

DEC-01
Zod for searchParams, not openapi-typescript. Search params are UI state, not backend contract. Zod gives runtime validation + type inference without coupling routes to API schema.
DEC-02
URL is derived state. Components never call router.navigate() directly. All navigation flows through app.machine events. The router is read-only from the component perspective.
DEC-03
fast-deep-equal guard prevents URL-sync loops. Without urlActuallyChanged: URL_CHANGED → syncURLToMachine → navigate → emit → infinite loop. The deep equality check breaks the cycle.
DEC-04
4 flat parallel regions max in app.machine. Currently at 5 (added atlasView). Sprint 2 should review whether focus can collapse into an overlay substate to return to 4.
DEC-05
Code-based routing only. No file-based plugin. Code-based gives full type inference at the route definition site and avoids magic file-system coupling. Cost is verbosity; benefit is debuggability.
